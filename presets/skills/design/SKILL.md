# design (글로벌 디자인 레퍼런스)

이 skill은 사용자의 `~/dev/design/` 디렉토리에 모인 디자인 자료를 조회하기 위한 가이드다. (BLUEPRINT §10)

## 위치와 구조

- 루트: `~/dev/design/`
- 자유 형식. 사용자가 자료를 그냥 떨어뜨려 둔다. HTML 프로토타입, PNG 스크린샷, .md 노트, .pdf 시안 등.

## 언제 이 skill을 발동해야 하는가

**디자인 결정을 하기 전에 항상.** 구체적으로:

1. **새 프로젝트의 디자인 토큰을 잡기 전** — `~/dev/design/`에 사용자가 선호하는 색상·타이포·레이아웃 자료가 있는지 먼저 확인.
2. **단일 컴포넌트 시안을 작업할 때** — 비슷한 컴포넌트 시안이 이미 있는지.
3. **사용자가 "디자인이 다르다"고 피드백할 때** — 그 즉시 `~/dev/design/`을 다시 본다. 사용자가 떨어뜨려 둔 시안이 진짜 의도일 가능성이 높다.

## 어떻게 사용하는가

```
1. Bash("ls ~/dev/design")  # 모든 자료 목록 (확장자 무관)
2. .html / .md / .txt 파일은 Read로 직접 읽음
3. .png / .jpg 같은 이미지는 Read로 시각 확인 가능 (Claude Code의 멀티모달 기능)
4. 발견한 자료를 designer 팀장에게 컨텍스트로 명시적으로 전달
```

## designer 팀장과의 관계

너(CEO 또는 다른 에이전트)가 designer를 호출할 때, **이 skill에서 본 자료의 경로를 designer 프롬프트에 명시적으로 포함**해야 한다. designer는 stateless 서브에이전트라 매번 컨텍스트로 전달되어야 자료를 본다.

예: `Task(designer, "~/dev/design/app_prototype.html을 시안으로 사용해서 디자인 토큰을 잡아라. ...")`.

## 갱신 정책

자료는 사용자가 직접 추가한다. 너는 **읽기만** 한다.

## 한 줄

디자인 의사결정 전에 `~/dev/design/`을 먼저 본다. 안 보면 사용자 의도와 어긋난다.
